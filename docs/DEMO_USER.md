## הרשאות משתמש הדגמה

תפקיד: **מנהלת המטה (admin)**

- **דשבורד מנהלת** — סטטיסטיקות, אישורים, בקרת נתונים
- **אישורים** — אישור/דחיית הרשמות (מרכזות וצוות מטה)
- **בקרת נתונים** — גישה לסניפים, חניכות, פעילויות, כספים
- **ניהול כספים** — הקצאות תקציב, אישור קבלות
- **ניהול משתמשים וסניפים** — רשימת סניפים, פרופיל סניף, חניכות וצוות
- **פתיחת / הוספת סניף** — הוספת סניף חדש
- **מלאי וציוד** — טבלת מלאי מחסן
- **הודעות** — שליחה וצפייה
- **מעקב ופיקוח** — פעילויות ונתוני סניפים
- **ניהול חומרים** — העלאת חומרים, בקשות שוברות שגרה
- **ארכיון תוכן** — חומרים לפי קטגוריה
- **בטיחות סניפים** — אישור בקשות, דיווחי בטיחות
- **פניות** — תיבת דיווחי בטיחות
- **תיק רפואי** — רגישויות ואלרגיות של חניכות

**מסך הדגמה (`/demo`):** רק משתמש עם אימייל `demo@batmelech.local` רואה מסך עם כפתורים לכל המסכים ולכל הסניפים.

---

## שלב 1: יצירת המשתמש ב-Supabase

1. היכנסי ל-**Supabase** ופתחי את הפרויקט של "בת מלך".
2. בתפריט השמאלי — **Authentication** → **Users**.
3. **Add user** → **Create new user**.
4. מלאי:
   - **Email:** `demo@batmelech.local`
   - **Password:** `770770770`
5. **Create user**.

---

## שלב 2: עדכון הפרופיל (תפקיד + מאושר)

1. ב-Supabase — **SQL Editor** → **New query**.
2. פתחי את הקובץ: **`supabase/sql/10_demo_user_profile.sql`**
3. העתיקי את כל התוכן והדביקי בשאילתה.
4. **Run**.

אחרי ההרצה — למשתמש יש תפקיד **admin** וסטטוס **approved**.

---

## שלב 3 (אופציונלי): הזנת סניפים

אם טבלת `branches` ריקה ורוצים להזין את רשימת הסניפים:

1. **SQL Editor** → **New query**.
2. פתחי את **`supabase/sql/11_branches_seed.sql`**.
3. העתיקי והריצי (מומלץ להריץ **פעם אחת**; הרצה חוזרת תיצור כפילויות).

---

## שלב 4: התחברות באפליקציה

1. פתחי את האפליקציה (למשל `npm run dev`).
2. גשי ל־`/login`.
3. הזיני אימייל `demo@batmelech.local` וסיסמה `770770770`.
4. אחרי ההתחברות — ניווט אוטומטי למסך הדגמה (`/demo`), ומשם גישה לכל המסכים ולכל הסניפים.
